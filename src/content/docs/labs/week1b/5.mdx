:::caution[Make sure GIT is installed!]
Before you can commit, you need to make sure that you have `Git` installed. If you don't have it installed, please follow the instructions [here](../../help/gitscm).
:::

**Commits** are the basis of `Git` and `GitHub`.

:::danger[Do not upload manually!]
While it may be tempting to manually upload files through GitHub, for the purposes of this class, this will cause many headaches. For this class you should **only** commit through VS Code!!!
:::

In a nutshell, **Committing** is `Git` speak for **saving files**. However, these saves come with a note so that you can refer back to any save later!

It's better than autosaving because you have a description of what that save is!

Let's make our first commit:

Click on the `Source Control` icon in the activity bar in VS Code:

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit1.png" alt="" style="max-width:300px" />

Enter a note for your commit:

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit2.png" alt="" style="max-width:300px" />

Click the `checkmark` OR use the following shortcut:

- **PC:** `Ctrl+Enter`
- **MAC:** `Cmd+Enter`

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit3.png" alt="" style="max-width:300px" />

Next, click on the `...` for expanding the menu items:

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit4.png" alt="" style="max-width:300px" />

Finally, click on `Push` to upload your changes to GitHub:

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit5.png" alt="" style="max-width:300px" />

If you see the `VS Code periodically wants to run fetch` dialogue, click `Yes` to allow it to randomly update your repository.

<img src="/26SU-ASIAAM-191A/labs/week1/media/vs_code_fetch.png" alt="" style="max-width:300px" />

### Optional: Easier Commits

:::tip[Timesaving Tip!]
You can avoid having to do the extra steps of clicking `...` then `push` by turning on a setting to `push` after each commit!

Here's how to do so:

- Go to `File` → `Preferences` → **Settings**

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit_menu.png" alt="" style="max-width:300px" />

- Search **post commit** in the text box at the top

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit_search.png" alt="" style="max-width:300px" />

- Change the setting from `none` to **push**

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit_setting.png" alt="" style="max-width:300px" />

- Now the next time you commit by pressing the following shortcut, it will push automatically!
  - **PC:** `Ctrl+Enter`
  - **MAC:** `Cmd+Enter`

<img src="/26SU-ASIAAM-191A/labs/week1/media/commit_box.png" alt="" style="max-width:300px" />
:::

## Using GitHub Pages

After you have saved and committed your files to GitHub visit your repository on https://www.github.com.

:::note[What is a repository?]
A repository is GitHub speak for a location for a specific project and/or bunch of files. You can think of it as a `folder`. For this class, instead of making a new repository for each week we will have just one lab assignment repository with **different** folders for the weeks. Learn more: [About repositories (GitHub.com)](https://docs.github.com/en/repositories/creating-and-managing-repositories/about-repositories).
:::
