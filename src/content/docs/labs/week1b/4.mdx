### Fixing a footer bug from Week 1

Last week, the `footer` was defined as an **ID** in the HTML (`<div id="footer">`), but selected in the CSS without an `#` ID selector:

```css title="styles/style.css"
/* ❌ This won't work — targets a <footer> element, not our <div id="footer"> */
footer { 
    grid-area: footer;
}
```

To fix this, add the `#` ID selector:

```css title="styles/style.css"
/* ✅ This targets our <div id="footer"> */
#footer {
    grid-area: footer;
}
```

## 🎨 Styling the Portfolio

With the list added, you may notice the bullet points are misaligned — they appear off to the side of the portfolio box.

### Not a pad(ding) idea!

A simple fix is to add `padding-left` to the `ul` element:

```css title="styles/style.css"
ul {
    padding-left: 30px;
}
```

But since we're using CSS Grid, we can do better by making the portfolio itself a flex container:

```css title="styles/style.css"
.portfolio {
    grid-area: portfolio;
    display: flex;           /* makes children flex items */
    flex-direction: column;  /* stacks children vertically */
    justify-content: center; /* centers content vertically */
    align-items: center;     /* centers content horizontally */
}
```

:::note[What the `flex`?!]
Flexbox (`display: flex`) is the predecessor to CSS Grid. While CSS Grid handles **2D layout** (rows and columns), Flexbox handles **1D layout** — distributing space among items in a single row or column. It's great for centering content inside a container.

Read more on MDN: [CSS Flexbox](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Flexible_Box_Layout/Basic_Concepts_of_Flexbox)
:::
