Basics of JavaScript

In Week 1, we covered HTML and CSS — the structure and style of a webpage. This week we add JavaScript, starting with variables — the building blocks of all JavaScript code.

Recall from the pre-lab reading that a webpage is like a house:

  • HTML is the scaffolding/foundation of the house
  • CSS is the paint, carpets, etc. that makes the house look nice
  • JavaScript is the appliances that adds function to the house

What is JavaScript?

JavaScript is like the scriptwriter behind the scenes, turning static web pages into dynamic, interactive stories that captivate the audience. While there are various frameworks like React.js and Vue.js that offer additional features, this class will focus on the basics of JavaScript, also known as “vanilla” JavaScript, following the ES6+ standards. Learn more about ES6+ standards here.

In HTML, we use <script> tags to include JavaScript code. These tags can be placed in two main spots:

  1. Inside the <head> section: JavaScript here loads before anything else on the page. Good for libraries like MapLibreGL that need to be ready before the page shows.
  2. Right before the closing </body> tag: This is where you put JavaScript that interacts with the page content — since it loads after the HTML, all your page elements exist before the script tries to use them.

Remember, the order in which scripts are loaded can make or break your webpage — imagine putting the ending of a mystery movie at the beginning!

It’s a me Lego Var-io

Variables are like boxes that hold information — think of them as the building blocks of instructions.

Variables can hold numbers, text, or even collections of other variables! In programming we call these types. To use a variable, you have to declare it. In JavaScript, variables are automatically assigned types based on what you put in them.

In JavaScript, all declarations and lines should end with a semicolon ;, which is like a . (period) in English — it says “my statement is done.”

What is a keyword?

In most coding languages, a keyword is a word that tells a program to treat the following text, numbers, or characters in a specific way. For example, var myName says treat myName as a variable. This means you CANNOT name a variable var — Jar Jar Binks’ cousin Var Var Binks is VARy bad for JavaScript to see! (var var = 💥) Also note: you cannot use spaces in variable names!

Some more variable definitions

Variables act like boxes where you can store or take information:

  • const acts like a locked safe — you can’t change what’s inside after you define it
  • let is like a regular box — you can swap out the contents
  • var is VARy problematic — it can be both locked and unlocked depending on where it’s declared

LET’s a go!

With JavaScript ES6, let and const were introduced to replace var. The recommended practice is to no longer use var. Both let and const are declared the same way:

Let vs Const vs Var

What is the difference?

  1. let — the variable can change (mutable)
  2. const — the variable cannot change (constant)
  3. var — can be either depending on where it was declared. VERY PROBLEMATIC!

Because var causes confusion by being both mutable and immutable depending on context, it was split into let and const.

Scopes: Local vs. Global

Where you declare a variable sets its scope — either local (limited to a function or block of code) or global (accessible anywhere). This is why var is dangerous: its scope rules are inconsistent.

So, bye bye var and LET us welcome our new CONST variables!

TLDR

DO NOT USE var unless you specifically need to support Internet Explorer.
